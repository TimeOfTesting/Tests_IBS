В ходе проведения тестирования было выявлено:
1. При запуске TestSuiteUI элемент span[data-key="response-code"](RESPONSE_STATUS)'
меняет свое значение после того как загрузится элемент pre[data-key="output-response"](RESPONSE_BODY),
до этого момента значение статуса в элементе span[data-key="response-code"] остается с предыдущего ответа.
Что иногда приводит к некорректному отражению статуса кода в элементе с правильным статусом кода от API.
По логике статус кодf должен обновляться одновременно с получением данных тела ответа.
При единичном запуске каждого теста данный баг не воспроизводится.
2. При тестировании API в эндпоинте api/users/{user_id} отсутсвует информация о имени и работе в теле ответа.
3. При тестировании поля ввода денежной поддрежки сайта, было выявлено, что поле принимает отрицательные значение
и ноль, далее перекидывает на страницу оплаты и указывает там значение по умолчанию(15$).
